NMAP - Network Mapping / SCANNING

NMAP Port Scan

To perform a basic port scan using Nmap, you can use the following command:

nmap [target]

Replace [target] with the IP address or hostname of the target system you want to scan.

For example:

nmap 192.168.1.1

This command will perform a default scan on the target system, identifying open ports and services.

If you want to specify a range of ports or a specific set of ports, you can use the -p option. For instance, to scan ports 80, 443, and 8080, you can use:

nmap -p 80,443,8080 192.168.1.1

Nmap also provides various scan options for more advanced scanning techniques. Here's an example of a more comprehensive scan:

nmap -p 1-1000 -sV -sC 192.168.1.1

In this example:

  • -p 1-1000: Specifies a range of ports from 1 to 1000.
  • -sV: Enables service version detection.
  • -sC: Runs default scripts to discover additional information.

SCANNING

1. Host Discovery

Host discovery is the process of identifying and determining the availability of active hosts (devices or systems) on a network. In the context of network scanning and security assessments, host discovery is a fundamental step to understand the scope of a network, identify live systems, and gather information about potential targets.

a. ARP PING SCAN

ARP (Address Resolution Protocol) Scans: On local networks, ARP can be used to discover hosts by sending ARP requests and listening for responses.

To perform a ARP PING SCAN using Nmap, you can use the following command:

nmap -sn -PR [IP_ADDRESS]

For example:

nmap -sn -PR 192.168.1.1

In this example:

  • -sn: This option instructs Nmap not to perform a port scan. It essentially tells Nmap to only conduct host discovery without going into the detailed examination of open ports.
  • -PR: This command performs an ARP ping scan (-PR).

b. UDP PING SCAN

A UDP ping scan is a method used by Nmap to discover live hosts on a network by sending UDP packets and listening for responses.

To perform a UDP PING SCAN using Nmap, you can use the following command:

nmap -sn -PU 192.168.1.1

In this example:

  • -sn: This option instructs Nmap not to perform a port scan. It essentially tells Nmap to only conduct host discovery without going into the detailed examination of open ports.
  • -PU: This command performs an UDP ping scan (-PU).

c. ICMP ECHO PING SCAN

An ICMP Echo (ping) scan is a straightforward method of host discovery in which the scanning tool (like Nmap) sends ICMP Echo Request packets to potential hosts and listens for ICMP Echo Reply responses. This type of scan is commonly used to quickly determine if hosts are online.

To perform a ICMP ECHO PING SCAN using Nmap, you can use the following command:

For example:

nmap -sn -PE 192.168.1.1

In this example:

  • -sn: This option instructs Nmap not to perform a port scan. It essentially tells Nmap to only conduct host discovery without going into the detailed examination of open ports.
  • -PE: This command performs an ICMP ECHO PING SCAN

ICMP echo ping sweep

This command will send ICMP Echo Request (ping) messages to each IP address in the specified range and listen for ICMP Echo Reply responses. It's a quick way to discover live hosts on a network.

Keep in mind that some hosts or networks may have firewalls or other security measures in place that block ICMP ping requests, so the absence of a response does not necessarily mean the host is down.

For example:

ICMP timestamp ping scan

An ICMP timestamp ping scan is a specific type of network reconnaissance conducted using ICMP (Internet Control Message Protocol) timestamp requests. This scan is typically performed using tools like Nmap. The primary goal of this scan is to determine the uptime and system clock information of a target host

For example:

nmap -PE -PP --reason 192.168.1.0

In this example:

  • -PE: This command performs an ICMP ECHO PING SCAN
  • -PP: This command performs an ICMP TIMESTAMP PING SCAN

ICMP ADDRESS MASK PING SCAN

An ICMP Address Mask Ping Scan is a type of network reconnaissance that uses ICMP (Internet Control Message Protocol) address mask requests to gather information about target hosts on a network. This type of scan is less common than ICMP echo requests (ping) or ICMP timestamp requests, but it serves a similar purpose in the host discovery process.

In an ICMP Address Mask Ping Scan, the scanning tool sends ICMP address mask requests to the target hosts. These requests ask the target systems to respond with their subnet mask information. By analyzing the responses, the scanning tool can gain insights into the network topology and potentially identify misconfigured or unusual subnet mask settings on target hosts.

For example:

nmap -PE -PM --reason 192.168.1.0

In this example:

  • -PE: This command performs an ICMP ECHO PING SCAN
  • -PM: This command performs an ICMP ADDRESS MASK PING SCAN

TCP SYN PING SCAN

A TCP SYN Ping Scan is a type of network reconnaissance technique that uses TCP SYN packets to discover hosts on a network. In this scan, the tool sends a TCP SYN packet (half-open connection) to the target host. If the target host responds with a TCP SYN-ACK packet, it indicates that the port is open, and the host is considered alive. If the target responds with a TCP RST (reset) packet, the port is closed.

For example:

nmap -sn -PS --reason 192.168.0.0/24

In this example:

  • -PS: This option enables TCP SYN (half-open) ping scan. It sends TCP SYN packets to the specified ports for host discovery.

TCP ACK Ping Scan

A TCP ACK Ping Scan is a type of network reconnaissance that uses TCP ACK (acknowledgment) packets to discover hosts on a network. In this type of scan, the tool sends TCP ACK packets to the target hosts. The response (or lack of response) can provide information about the state of the ports on the target host.

In a TCP ACK Ping Scan, the tool sends TCP ACK packets to the target ports, and the responses can provide information about the state of those ports. This type of scan can be useful for detecting hosts that might be silently dropping packets or identifying hosts that have certain ports open.

For example:

nmap -sn -PA --reason 192.168.0.0/24

In this example:

  • -PS: This option enables TCP ACK (acknowledgment) ping scan. It sends TCP ACK packets to the specified ports for host discovery.

IP PROTOCOL PING SCAN

An IP Protocol Ping Scan, or simply an IP Protocol Scan, is a network reconnaissance technique that involves scanning a target system to identify the supported network-layer protocols. In this type of scan, the tool sends packets with different IP protocols (other than the default ICMP) to the target system, and based on the responses or lack thereof, it determines which IP protocols are supported.

Keep in mind that not all systems respond to IP protocol scans, and the effectiveness of this type of scan can depend on the target's firewall settings and network configuration.

For example:

nmap -PO --reason 192.168.0.0/24

In this example:

  • -PO: : This option enables IP protocol scan. It sends packets with different IP protocols to the target system.

2 PORT AND SERVICE DISCOVERY

Port and service discovery is a crucial aspect of network reconnaissance, helping identify which services are running on a target system and which ports are open. Various tools, including Nmap, are commonly used for this purpose.

TCP Connect Scan/FULL-OPEN SCAN

A TCP Connect Scan, often referred to as a full connect scan, is a basic and straightforward port scanning technique used to determine which TCP ports on a target system are open, closed, or filtered. This scan type is one of the simplest and most detectable methods, as it involves completing a full three-way TCP handshake for each port being scanned.

For example:

nmap -sT -v 192.168.0.105

In this example:

  • -sT: : his option specifies a TCP Connect Scan, which involves completing a full three-way TCP handshake for each scanned port. It is more detectable than other scan types but provides accurate results.
  • -v: : This option enables verbose output, providing more detailed information about the scan progress and results.

STEALTH TCP SCANNING

STEALTH TCP SCANNING/ half open scan

Stealth TCP scanning, also known as stealth scanning or half-open scanning, is a technique used in network reconnaissance to scan for open ports on a target system while attempting to minimize the chances of detection by intrusion detection systems (IDS) or firewalls. The primary goal is to gather information about open ports and services without completing the full three-way TCP handshake.

It's important to note that while SYN Stealth Scans are stealthier than TCP Connect Scans (-sT), they are not completely undetectable. Advanced firewalls and intrusion detection systems may still detect such scans.

For example:

nmap -sS -v 192.168.0.105

In this example:

  • -sS: This option specifies a SYN Stealth Scan, which is a stealthy and commonly used scan type.
  • -v: : This option enables verbose output, providing more detailed information about the scan progress and results.

STEALTH TCP SCANNING/ INVERSE TCP FLAG SCAN

In general, TCP flags in a TCP packet include options such as SYN (synchronize), ACK (acknowledge), FIN (finish), RST (reset), PSH (push), and URG (urgent), among others. Scanning techniques involve manipulating these flags to gather information about the state of ports on a target system.

STEALTH TCP SCANNING/ INVERSE TCP FLAG SCAN / FIN scan

A FIN scan is a type of TCP port scan used in network reconnaissance. In this scan, the scanning tool sends TCP packets with the FIN (finish) flag set to the target system's ports. The FIN flag indicates the end of a TCP connection, and an open port should respond to the FIN packet with a TCP RST (reset) if the port is closed, and it should ignore the packet if the port is open.

STEALTH TCP SCANNING/ INVERSE TCP FLAG SCAN / NULL scan

A NULL scan is a type of TCP port scan used in network reconnaissance. In a NULL scan, the scanning tool sends TCP packets with no flags (NULL flags) set to the target system's ports. The NULL flag means that none of the TCP control flags (FIN, SYN, RST, PSH, ACK, URG) are set in the TCP header.

In a NULL scan, the behavior of the target system can reveal whether a port is open or closed. If a port is closed, it should respond to a NULL packet with a TCP RST (reset). If a port is open, it should not respond at all.

Here's an example of how to perform a NULL scan using Nmap:

nmap -sN -p 1-65535 192.168.0.105

STEALTH TCP SCANNING/ INVERSE TCP FLAG SCAN / ACK FLAG PROBE SCAN

Attackers send TCP probe packets with the ACK flag set to a remote device and then analyze the header information (TTL and WINDOW field) of the received RST packets to find out if the port is open is closed.

The ACK flag probe scan exploits the vulnerabilities within the BSD-derived TCP/IP stack. Thus, such scanning is effective only on those OSs and platforms on which the BSD derives TCP/IP stacks.

PORT SCANNING / UDP SCANNING

UDP scanning is a technique used in network reconnaissance to identify open UDP (User Datagram Protocol) ports on a target system. Unlike TCP, UDP is a connectionless protocol, and scanning UDP ports can be more challenging because of the lack of a three-way handshake and the non-guaranteed delivery of UDP packets.

Open UDP Port:

  • 1. If a UDP port is open, the target system may respond with a UDP packet, and Nmap considers the port as open.
  • 2. No response or an ICMP port unreachable message may also indicate an open port.

Closed UDP Port:

  • 1. If a UDP port is closed, the target system may respond with an ICMP port unreachable message.
  • 2. No response can also indicate a closed port.

Here's an example of how to perform a basic UDP scan using Nmap:

nmap -sU -v -p 1-65535 192.168.0.105

In this example:

  • -sU : This option specifies UDP SCANNING
